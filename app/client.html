For new users

If you're new to using IRC, welcome, and we're honoured that you chose Ave to be your first IRC chat client (and hopefully your only one!). Of course, though, to help allay any confusion that might arise, we advise reading the following information so that you know what some terms mean, and where you can get help with using Ave.

Firstly, if you're in need of help, checking the wiki is usually a good idea, as it contains information both valuable to the use of Ave for new users, and some information for developers (for new users, we recommend reading the Getting Started page). If the wiki doesn't help you, however, we recommend either opening on issue on GitHub, or asking in the #ave-irc channel on Freenode (IP irc.freenode.net, port 6667).

Also, why not take a quick look at the jargon buster below, designed to help you know exactly what we're talking about throughout the client?

Terminology Description
Chat Channels Chat channels are a system in IRC to help organise and separate conversations regarding different topics and
projects. Each chat channel, which is either an IRC channel (a chat room which groups of people can join and
leave; conversations here are public) or a private chat (a one-to-one discussion between two people, which are
usually private) is shown as a separate tab in the tabs bar at the top of the client. (Please be warned that private
chats can be made public if the person whom you're talking to is logging the conversation, and releases the logs
to the public. Be careful when chatting online.)
Nickname In IRC, each user is identified by a nickname (or "nick" for short). Thus,
only one user can have a particular nickname at one time. (Think of nicknames as usernames in most
other online services.)

What makes Ave different

Ave is, in many ways, different to most of your usual IRC clients. Part of this is because Ave's design is inspired, uniquely for an IRC client, by Google's Material Design Guidelines. But Ave is also unique in that user interface is valued above command interfaces.

It is common for IRC clients to implement commands via the textual input fields, resulting in users entering /msg nickserv identify password to identify with NickServ. We working on Ave, however, consider this bad form as users should not be expected to memorise a long list of commands and their syntaxes.

Consequently, Ave aims to implement as many commands as possible naturally into the client's user interface, allowing users to do what they can do in other clients with a logical, and natural, graphical user interface.

(An example is that joining channels, in other clients, is usually done with /join #channel. In Ave, this is accomplished by opening the side menu (click the hamburger icon in the top left), clicking "Join IRC Channel", entering the channel name and clicking "join". It sounds like a lot - we know - but it's not (and actually a lot more natural!).)
